# Collab / invites : masquer UI v0, cible collab async v1

## Context

Le blueprint exige **zéro promesse fantôme**. L’UI expose `ProjectMembersPanel` + i18n `members.*` (invite editor/viewer) alors que `docs/prd/PRD.md` exclut la collaboration multi-utilisateur en v0.

## Decision

| Option | Choix |
|--------|--------|
| A — Masquer invites en prod externe | **Oui, immédiat (impl hors ce doc)** |
| B — Spec collab async v1 | **Oui, documentée mois 1** (`docs/TODO.md` — collab scope slice) |
| C — Livrer invites editor/viewer v0 | **Non** |

### Court terme (semaine 1–2, produit)

- Ne plus afficher le panneau **Team members** sur le workspace projet tant que la slice collab n’est pas **shipped** en prod.
- **Statut 2026-06-04 :** panneau retiré du dialog Settings projet (`project-workspace.tsx`) — alignement doc/code.
- Conserver les clés i18n pour éviter churn technique ; pas de copy utilisateur visible ailleurs.

### Cible v1 (mois 1, spec séparée)

- **Commentaires async par section** (pas co-édition live).
- Rôles : owner, commenter (invité par email), viewer share inchangé.
- Pas d’édition concurrente du PRD en v1.

## Consequences

**Pros :** alignement confiance PRD v0 ; pas de support « l’invite ne marche pas comme Notion ».

**Cons :** fondateurs en équipe doivent utiliser **share link** + commentaires externes jusqu’à v1.
