## Zedos v0 + Hexagonal Architecture Refactor

### Overall Vision

**Zedos** is a web product helping solo founders move from vague product ideas to versioned PRDs through guided AI clarification. The codebase has been refactored from a flat structure into a **hexagonal (clean) architecture** — production-grade: testable, maintainable, and migration-ready to monorepo.

### Architecture: Hexagonal (6 Layers)

```
nextjs_space/
├── src/
│   ├── domain/              # Pure business logic, zero external deps
│   ├── application/         # Use-cases, orchestration
│   ├── contracts/           # Zod schemas (single source of truth for I/O)
│   ├── infrastructure/      # Adapters: Prisma repos, wrapped SDKs (Stripe, AI, Auth)
│   ├── ui/                  # React components, isolated from logic
│   └── shared/              # Result<T,E>, errors, logger, events, mappers
├── app/                     # Next.js routes (thin HTTP adapters)
├── components/              # (Legacy - migrate to src/ui/)
├── lib/                     # (Legacy - migrate to src/application/ or infrastructure/)
└── prisma/                  # Database schema
```

### Key Principles

1. **Result<T,E> Everywhere** — No exceptions crossing boundaries.
2. **Thin Routes** — API routes are ~10 lines: parse → call use-case → map to HTTP.
3. **SDK Wrapping** — Every external vendor wrapped in infrastructure, validates via Zod.
4. **Contracts = Source of Truth** — Single Zod schema per data structure.
5. **Layer Boundaries Enforced** — ESLint `boundaries` plugin at `error` level; TypeScript path aliases.
6. **Testable** — Business logic decoupled from frameworks; 65 unit tests via Vitest.

### ESLint Boundary Enforcement

- `element-types` rule at `error` level for `src/` files (6 layer rules: domain→shared only, application→domain+contracts+shared, etc.)
- `element-types` rule at `error` level for `app/` files (app → all layers + legacy)
- `no-unknown` at `error` for `src/`, `warn` for `app/` (path alias limitation with `@/lib`)
- Legacy `lib/` and `components/` have `element-types` off
- Run: `ESLINT_USE_FLAT_CONFIG=false npx eslint src/ app/`
- Note: plugin v6 deprecation warnings for `element-types` → `dependencies` rename are benign

### Current v0 Feature Set

✅ Auth — Email/password signup + login (via NextAuth)
✅ Projects — Create, view, list, delete
✅ PRD Versions — Structured sections (8 core: overview, problem, users, journeys, objects, scope, risks, metrics)
✅ Guided Clarification — Chat-driven AI questions (single_choice, multi_choice, ranked, custom input)
✅ Credits — 20 starter, tiered burn (1/3/5/10/15), grace period (overage up to 20 only once)
✅ Credit Packs — 100/€19, 200/€29, 1000/€99 (EUR only, Stripe checkout)
✅ Question History — Structured log of all clarifications
✅ Share Links — Read-only public PRD shares (revocable, noindex)
✅ Milestone Feedback — 4 milestones, 5-star + comment, skippable
✅ ADRs (L1+L2) — 14 slots, phase-gating (intake → architecture), readiness score (Product Clarity 10pts, Architecture Boundaries 15pts)
