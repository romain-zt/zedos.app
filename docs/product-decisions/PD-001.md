# Delivery export format: `.cursor/` folder structure + WORK_QUEUE-compatible entries

## Context

The Delivery Feature Area (FG-POST-PRD-V1) packages the output of the Test-first workflows area (user stories + tasks + prompts) into a Cursor-ready deliverable. The exact export format was unresolved and marked `NEED_HUMAN: true`, blocking Scope Slice definition and validation.

Two realistic options existed:
1. Generic markdown files in a `docs/` folder — portable but not Cursor-native.
2. `.cursor/` folder structure + `WORK_QUEUE.md`-compatible entries — matches Cursor's own conventions and what Zedos uses internally.

The PRD already names option 2 as the initial target (`FG-POST-PRD-V1 Sub-components: "Package and export user stories + task prompts into a Cursor-ready format; initial target: .cursor/ folder structure + WORK_QUEUE-compatible entries"`).

## Decision

**Export format: `.cursor/` folder structure + `WORK_QUEUE.md`-compatible entries.**

Concretely, the Delivery package contains:
- A `WORK_QUEUE.md` in the format Zedos uses internally — rows for each user story / task with ID, status, and next action fields.
- Per-story files (e.g. `docs/execution/user-stories/<story>.md`) with actor, outcome, acceptance criteria.
- Per-task prompt files or inline task prompts embedded in the story files (one structured prompt per task, ready for Cursor agent consumption).

The founder downloads this package (ZIP or folder) and drops it into their repo. A Cursor agent can immediately start working through the `WORK_QUEUE` without any reformatting.

## Consequences / tradeoffs

**Pros:**
- Self-referential: the output of Zedos matches how Zedos itself is governed — the founder understands the format by using the product.
- Zero custom format to maintain: reuses existing Zedos templates and conventions.
- Directly consumable by Cursor agents out of the box.
- Validates the product's own workflow (dogfooding).

**Cons:**
- Coupled to Cursor's conventions; founders using other AI coding tools get less native integration (acceptable for v1 — Cursor is the primary target per PRD).
- If Zedos internal governance format evolves, delivery format may need to track it (manageable — both are under Zedos control).
