# Marketing

Use for positioning, GTM, landing pages, and growth. Pairs with `copywriter` for the words.

## Start with positioning (before tactics)

Fill these before writing any asset:

- **ICP:** who specifically (role, context, trigger). Not "everyone".
- **Problem:** the painful job-to-be-done, in their words.
- **Alternatives:** what they use today (incl. "do nothing").
- **Differentiation:** the one thing you do better that they care about.
- **Proof:** evidence (numbers, demos, testimonials) — not adjectives.

## Messaging hierarchy

One primary message → 3 supporting points → proof per point. Same hierarchy drives the landing page, ads, and onboarding. Keep claims honest and specific.

## Landing page skeleton

Hero (problem + outcome + one CTA) → how it works (3 steps) → proof → objections/FAQ → final CTA. Mobile-first (see `design`). One primary CTA per page.

## Funnel & channels

- Map awareness → consideration → activation → retention; define the one metric per stage.
- Pick 1–2 channels to do well, not all of them. Match channel to where the ICP already is.
- Every campaign states a hypothesis, a metric, and a kill/scale criterion.

## Metrics (measurable, not vibes)

Define success numerically: activation rate, CAC, conversion, retention. "Adoption grows" / "users are happy" are not metrics.

## Output

```txt
GTM note — <initiative>

ICP: <who + trigger>
Problem → outcome: <one line>
Primary message: <one line> · supports: <3> · proof: <per support>
Channel(s): <1–2 + why>
Funnel metric: <stage → metric → target>
Experiment: hypothesis · metric · kill/scale
```

## Anti-patterns

- Tactics before positioning. Targeting "everyone".
- Feature lists instead of outcomes. Adjectives instead of proof.
- Vague success criteria; spreading across every channel at once.
